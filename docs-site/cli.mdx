```text
Tunelito 0.21.1

Usage: tunelito <page.html|notes.md|folder> [options]
       tunelito doctor [page.html|notes.md|folder] [options]
       tunelito mcp
       tunelito comments inspect <page.html|notes.md|folder|comments.md> [options]
       tunelito review watch [page.html|notes.md|folder] [options]
       tunelito inbox <next|watch|status|record> <page.html|notes.md|folder> [options]
       tunelito session status [page.html|notes.md|folder] [options]
       tunelito config show [page.html|notes.md|folder] [options]
       tunelito skill <show|setup|install>

Coding agents:
  Before serving a review room, load the bundled workflow:
    tunelito skill show
  Installation guidance:
    tunelito skill setup

Session model:
  Default               Persistent Markdown comments plus browser hot reload
  Real-time sync        WebSocket in every session; WebRTC peers with --ephemeral
  --ephemeral           In-memory comments that are lost when the server stops
  --agent-session       Current coding-agent conversation watches persistent feedback
  --agent               Spawn a separate local coding-agent worker
  Tunnel exposure       Enabled by default; --no-tunnel keeps the room local

Options:
  --port <number>       Port to listen on (default: first free from 4317)
  --host <host>         Host to bind locally (default: 127.0.0.1)
  --out <path>          Markdown comments file (default: <page-or-folder>.comments.md)
  --theme <name>        Markdown theme: default|editorial|technical|bns-pitaya
  --markdown-css <href> Add a stylesheet link to rendered Markdown pages
  --owner <name>        Seed the editable owner name for the direct local viewer
  --ephemeral           Keep comments in memory only; all feedback is lost on restart
  --live                Deprecated alias for --ephemeral
  --agent <codex|claude|custom>
                        Run a local coding-agent worker for persistent comments
  --agent-command <cmd> Custom shell command for --agent custom; prompt is sent on stdin
  --agent-interval <s>  Agent fallback polling interval in seconds (default: 120)
  --agent-policy <mode> Which comments are actionable: all|mention|owner|owner-or-mention (default: all)
  --agent-trigger <txt> Marker for mention policies, or "all" (default: all)
  --agent-instructions <txt>
                        Append host instructions to the built-in agent prompt
  --agent-instructions-file <path>
                        Append host instructions from a file
  --agent-prompt <txt>  Replace the built-in agent behavior prompt
  --agent-prompt-file <path>
                        Replace the built-in agent behavior prompt from a file
  --agent-max-attempts <n>
                        Stop retrying a comment after n attempts (default: 2)
  --agent-max-passes <n>
                        Stop continuing a multi-pass comment after n agent passes (default: 3)
  --agent-state <path>  Agent resolution ledger (default: <target>/.tunelito/agent/state.json)
  --agent-session       Watch comments for the current agent session; do not spawn a worker
  --no-tunnel           Only print the local URL; do not start Cloudflare Tunnel
  --no-auth             Disable the generated review-key URL gate
  --open                Open the local URL in your default browser
  -v, --version         Show version
  -h, --help            Show this help

Commands:
  doctor                Run read-only local setup and safety diagnostics
  mcp                   Start a stdio MCP server for comments and inbox tools
  comments inspect      Print a structured JSON index for a Tunelito comments inbox
  review watch          Wait for a browser Done Reviewing handoff event
  inbox next            Claim the next pending comment and print an agent prompt
  inbox watch           Wait for the next pending comment, then print an agent prompt
  inbox status          Print a live to-do tracker from the comments inbox and ledger
  inbox record          Record the active agent's result for one comment
  session status        Inspect and recover the last session for a target
  config show           Print resolved Markdown configuration and its source layers
  skill show            Print the distributable Tunelito agent skill (SKILL.md)
  skill setup           Print no-write setup guidance for common coding agents
  skill install         Install the skill for Codex or Claude at user or project scope
```

## Common examples

Start a public review session:

```bash
tunelito ./page.html
```

Serve a Markdown memo with the built-in readable page style:

```bash
tunelito ./notes.md
```

Choose a bundled Markdown theme:

```bash
tunelito ./notes.md --theme editorial
```

Persist themes and CSS with project or global configuration, then inspect the resolved layers:

```bash
tunelito config show ./notes.md
tunelito config show ./notes.md --json
```

See [Markdown themes and configuration](/configuration) for file locations, precedence, and the config schema.

Fenced `mermaid` blocks render automatically from Tunelito's packaged same-origin runtime. Other fenced languages stay as source code, and invalid diagrams keep a readable source fallback.

Complete HTML comments are hidden from the rendered reader surface while literal comment syntax inside code stays visible. In folder mode, the collapsible left sidebar labels its injected served-document tree as `Tunelito navigation`; root documents stay visible and nested folders start closed. Leading YAML front matter renders below it as a separate source-derived Properties section, which is omitted when a note has no front matter. Folder URLs without an authored index receive a themed landing page with immediate documents, child folders, and safe parent navigation. Common Obsidian wiki references such as `[[Note]]`, `[[Note|Alias]]`, and heading variants render as unresolved inline references without inventing navigation targets. On desktop, a compact, vertically centered right-edge document-map dial marks the real rendered headings and content blocks; it expands while hovered or visibly keyboard-focused and retreats when the pointer leaves. h1 through h6 remain keyboard-navigable, while paragraph ticks show reading progress as you scroll. These review-only surfaces support light and dark mode, reduced motion, and the comments panel without changing the source Markdown.

The durable Markdown fixtures cover minimal prose, long documents, front-matter edge cases, heading hierarchy, ruler density, Mermaid, and folder-style wiki references:

```bash
tunelito ./examples/markdown/frontmatter-flat.md --no-tunnel --open
tunelito ./examples/markdown/heading-ladder.md --no-tunnel --open
tunelito ./examples/markdown-vault --no-tunnel --open
```

Add a team stylesheet to rendered Markdown pages:

```bash
tunelito ./notes.md --markdown-css ./review.css
```

Serve a folder of HTML and Markdown files:

```bash
tunelito ./site
```

Run local-only and open the browser:

```bash
tunelito ./page.html --no-tunnel --open
```

Run read-only local diagnostics before starting a session:

```bash
tunelito doctor ./page.html --json
```

Choose a port:

```bash
tunelito ./page.html --port 4400
```

Choose the comments file:

```bash
tunelito ./page.html --out ./notes/review.md
```

Inspect the comments inbox as JSON for an agent or diagnostic tool:

```bash
tunelito comments inspect ./site --json
tunelito comments inspect ./site --out ./notes/review.md --json
tunelito comments inspect ./site.comments.md --agent-state ./site/.tunelito/agent/state.json --json
tunelito comments inspect ./site.comments.md --json
```

Seed the local owner display name:

```bash
tunelito ./page.html --owner "Chekos"
```

Run an ephemeral in-meeting session:

```bash
tunelito ./page.html --ephemeral
```

Run Codex against review comments:

```bash
tunelito ./site --agent codex
```

Use the current agent session instead of spawning a child agent:

```bash
tunelito ./site --agent-session --owner "Chekos" --agent-policy owner-or-mention --agent-trigger "@agent"
```

The same process serves the room, watches comments, claims actionable feedback, prints a prompt for the current agent, and shows browser-visible status on each comment card. Record the current agent's result with the claim id from the printed command:

```bash
tunelito inbox record ./site --id c_... --claim claim_... --status resolved --summary "Updated hero copy." --file index.html
```

If a claim was taken by another foreground/background claimer in the same workspace, inspect it with `tunelito inbox status ./site` and record with `--claim auto` when you intentionally want to resolve the currently owning claim.

Print the same comment work tracker in the terminal:

```bash
tunelito inbox status ./site
```

Inspect or recover the served session from a second shell:

```bash
tunelito session status ./site
tunelito session status ./site --json
tunelito session status ./site --redact
```

The command verifies the listener's session identity, reports `running`, `degraded`, `stale`, or `stopped`, and includes local/public URLs, persistence, agent work, process, tunnel, viewer, and recovery details.

Wait for the reviewer to click `Done Reviewing` in the browser panel:

```bash
tunelito review watch --url "http://127.0.0.1:4317/?tunelito_key=..." --json --timeout 600
```

The handoff event is in-memory and retained only while the server is running. `review watch` replays retained events after sequence `0` by default; pass `--after latest` to wait only for a future click.

Start the MCP stdio adapter for MCP-capable agents:

```bash
tunelito mcp
```

The MCP server exposes comments index, pending feedback, claim, watch, record, and status tools. It does not start a review server, tunnel, browser, or local agent worker.

Print guided no-write agent setup instructions:

```bash
tunelito skill setup
```

Preview and install the bundled skill:

```bash
tunelito skill install --agent codex --scope user --dry-run
tunelito skill install --agent claude --scope project
```

Allow broad comments to continue for up to five passes:

```bash
tunelito ./site --agent codex --agent-max-passes 5
```

Run Claude Code against review comments:

```bash
tunelito ./site --agent claude
```

Use another local CLI:

```bash
tunelito ./site --agent-command "openclaw run --stdin"
```

Append host instructions to the worker prompt:

```bash
tunelito ./site --agent codex --agent-instructions "Use terse, production-ready copy."
```

Require an explicit marker for stricter sessions:

```bash
tunelito ./site --agent codex --agent-trigger "@agent"
```

Handle owner-authored or owner-approved comments automatically and visitor comments only when tagged:

```bash
tunelito ./site --owner "Chekos" --agent codex --agent-policy owner-or-mention --agent-trigger "@agent"
```

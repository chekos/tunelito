## Requirements

- Node.js 22 or newer
- An HTML file or folder of HTML files on your machine
- Internet access if you want a public tunnel URL

## Start a review

```bash
npx --yes tunelito ./page.html
```

When `cloudflared` is installed, or when `npx cloudflared@latest` can run, Tunelito prints a `Public:` URL.

Tunelito also prints the comments file path:

```text
Comments: /path/to/page.comments.md
```

Reviewers select text in the browser, click `Comment`, and Tunelito writes the note to that Markdown file. They can also use `Page note` for unanchored feedback on the current page or `Site note` for feedback that should appear on every page in a folder review.

To review a folder-backed mini-site, pass the folder instead:

```bash
npx --yes tunelito ./site
```

Folder reviews write one comments file beside the folder:

```text
site/
site.comments.md
```

Each folder comment includes a scope and page path, so a person or coding agent can map page feedback back to the right HTML file or treat site feedback as applying across the folder.

## Start a review with local agent follow-up

If Codex is installed and logged in locally:

```bash
npx --yes tunelito ./site --agent codex
```

If Claude Code is installed and logged in locally:

```bash
npx --yes tunelito ./site --agent claude
```

The worker uses the local CLI's existing authentication. Tunelito records handled comment IDs in `.tunelito/agent/state.json` so the same comment is not applied repeatedly.

## Local-only preview

Use this when you do not want to expose a public tunnel:

```bash
tunelito ./page.html --no-tunnel --open
```

## Try the bundled example from a clone

```bash
git clone https://github.com/chekos/tunelito.git
cd tunelito
npm install
npm link
tunelito ./examples/simple-review.html
```

## Stop the session

Press `Ctrl+C` in the terminal. Tunelito stops the local server and tunnel process.

---
title: "Comments"
description: "How Tunelito stores reviewer comments and restores live session state."
---

Reviewers can select text and leave anchored comments from the injected Tunelito panel. They can also add unanchored page notes or site-wide notes without selecting text. Comments sync live to connected browsers and are written to Markdown immediately.

On mobile, Tunelito keeps the always-visible control small: a compact speech-bubble button with a count badge sits above common bottom navigation. Tap it to open the comments sheet.

Use `--live` when comments should be ephemeral and disappear with the local server instead of being written to disk.

## Default file location

```text
page.html
page.comments.md
```

For a folder target, the default comments file is beside the folder:

```text
site/
site.comments.md
```

Use `--out` to choose a different path:

```bash
tunelito ./page.html --out ./review-notes.md
```

## Comment scopes

Tunelito stores a `scope` for every comment:

- `page`: feedback belongs to the current page. Selected-text comments and page notes use this scope by default.
- `site`: feedback belongs to the whole served folder. Site notes appear in the sidebar on every page in a folder review.

Site comments still record the page where they were created. That origin page gives a person or agent useful context, but the comment is visible across the folder session.

If a site comment is created from selected text, the quote is preserved for context and highlighted on the origin page only. Other pages show the sidebar entry without trying to attach that quote to unrelated text.

## Markdown format

The comment file is designed to be readable:

```markdown
# Tunelito comments for `page.html`

## Jane at 2026-05-22 18:00:00 UTC

> selected text

This sentence needs a clearer verb.

_Context: scope: `page` · page: `/about.html` · path: `body > main > p` · text offset: 42 · id: `c_...`_
```

Unanchored notes do not include a quote block:

```markdown
## Jane at 2026-05-22 18:05:00 UTC

_Site note (no selected text)._

Use the same heading rhythm across every itinerary page.

_Context: scope: `site` · page: `/day-03.html` · id: `c_...`_
```

Tunelito also stores hidden metadata comments. Do not delete that metadata if you want Tunelito to restore the live sidebar after a restart.

## Agent inbox workflow

The Markdown file can act as a local inbox for a coding agent. Keep Tunelito running in persistent mode, then ask Claude Code, Codex, or another local agent to periodically inspect the comments file, edit the matching source HTML for page comments, apply clearly relevant changes across the folder for site comments, and report which visible comment IDs were addressed.

Tunelito can also run that polling loop for you. By default, every persistent comment is sent to the worker and the local agent decides whether to edit, return `no-op`, or mark it `ignored`:

```bash
tunelito ./site --agent codex
```

The worker writes status to `.tunelito/agent/state.json` instead of editing the comments file. That state prevents already handled comment IDs from being retried after source edits make the original highlight stale.

Large actionable comments can continue across several bounded passes. When the agent returns `needs_followup`, Tunelito saves completed and remaining tasks in the state file and queues that same comment again. This applies to selected-text comments, page notes, and site notes.

For stricter sessions, ask reviewers to include `@agent` in comments they want the worker to handle:

```bash
tunelito ./site --agent claude --agent-trigger "@agent"
```

Manual prompt:

```text
Monitor site.comments.md every 2 minutes. For each new actionable comment, edit the matching HTML file and summarize the comment id plus the change made. For larger comments, keep track of completed and remaining tasks so the next pass continues instead of repeating work.
```

For a complete end-to-end example, see the [folder review tutorial](/tutorial) and the [local agent worker guide](/agent-worker).

## Ephemeral live mode

```bash
tunelito ./page.html --live
```

In live mode, comments are held in memory only. The sidebar updates during the session, but no comments file is written and nothing is restored after restart.

## Highlight limits

If you edit the exact text someone commented on, the comment remains in Markdown and in the sidebar, but the highlight may not reattach.

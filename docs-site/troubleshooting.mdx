title: "Troubleshooting"
description: "Fix common Tunelito install, tunnel, comment, and browser issues."
---

## No public URL appears

Tunelito starts Cloudflare Tunnel with installed `cloudflared` first, then falls back to `npx cloudflared@latest`.
It ignores Cloudflare's provisioning endpoint, waits for the generated Quick
Tunnel hostname, and verifies that the hostname reaches the current Tunelito
server before printing `Public:`.

If neither works:

```bash
tunelito ./page.html --no-tunnel
```

Then install or allow `cloudflared` before trying again.

## Unkeyed URL returns 401

That is expected. Use the full printed URL with `tunelito_key=...`.

## Comments are saved but highlights disappeared

The selected text probably changed. Comments remain in Markdown and the sidebar, but highlights may not reattach if the exact quote can no longer be found. Page notes and site notes have no selected quote, so they appear in the sidebar and Markdown without a highlight.

## Live mode says relay instead of P2P

That is expected on some networks. `--live` tries WebRTC data channels for cursors, selections, and instant comments, but keeps the authenticated WebSocket connection as a fallback relay when peer-to-peer connectivity is unavailable.

Tunelito does not configure third-party STUN or TURN servers.

## Mobile selection feels finicky

Mobile browsers expose selection events differently. Try selecting a shorter phrase, then tap the temporary `Comment` button once it appears. The compact speech-bubble button opens the comments panel but does not create a selected-text comment by itself.

## Home Screen links open a browser sheet

This is usually site metadata, not Tunelito. Add a web app manifest plus iOS Home Screen meta tags to the HTML folder, and keep internal links inside the manifest scope without `target="_blank"`. See the [iPhone Home Screen guide](/iphone-home-screen).

## Page JavaScript behaves differently

Tunelito removes strict in-page CSP meta tags from the served response so the injected same-origin client can run. It does not write that change back to disk.

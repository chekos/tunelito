---
title: "Pre-call checklist"
description: "A quick pre-call checklist for sharing Tunelito with another person."
---

Use this before sharing a live session.

1. Start with an HTML page that does not already include another review system.
2. Run:

   ```bash
   npx --yes tunelito ./page.html
   ```

3. Wait for the `Public:` URL.
4. Open the public URL on your phone.
5. Select text and submit a short comment.
6. Add a short page note or site note from the comments panel.
7. Confirm the terminal logs `Comment from ...`.
8. Confirm `<page>.comments.md` contains the comment with `scope: ...`.
9. Edit and save the HTML file.
10. Confirm the phone reloads.

For an ephemeral call where comments should not be written to disk, run `npx --yes tunelito ./page.html --live` and skip the markdown-file check.

## What to avoid

- sensitive documents you would not be comfortable exposing through a bearer-link tunnel
- pages where the important content is only canvas, video, image text, or cross-origin iframes
- long-lived sessions where you cannot monitor who has the link
